Demos - Case Study :: Pong - Intro

In order to show how one might approach a non-trivial project with TactJS, I decided to recreate Pong. This case study is split into several iterations which can be downloaded individually and serve to track the progress toward an optimal solution.

I'm aware that there are HTML5 techniques for accomplishing the objectives of this case study, and if you were really tasked with creating a game for the browser without a plugin like Flash I would suggest you look into them. The real purpose of this case study however, is to explore the possibilities of TactJS, and something like Pong lets us have some fun along the way.

The stated objectives of Case Study :: Pong are thus:

  1. Two players, one of which is an AI opponent.
  2. Track score per round.
  3. Difficulty levels, with increasing difficulty from round to round.
  4. 2d ball physics (optional)

The solution must also meet the following criteria:

  1. Object oriented project library.
  2. Front-end uses TactJS behaviors.
  3. Event and message driven contracts.

This case study is a work in progress. Please continue to check for updates to the iteration list. If you would like to try your hand at a solution using TactJS, by all means let me know what you come up with by sending your solution to pong-solutions@tactjs.com
